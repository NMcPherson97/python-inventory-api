displayBooks()

STEP 5 - Make the Read Status button interactive
------------------------------------------------------------------------------
Goal:
Your Read Status button already exists.

Now connect it to your Book objects.

Workflow:

Click Read Status
    ↓
Determine which book was clicked
    ↓
Call that Book's toggleReadStatus() method
    ↓
displayBooks()

Do NOT manually change the card text.

Update the object first, then re-render the UI.

Review:

------------------------------------------------------------------------------
STEP 6 - Give each Book its own behavior
------------------------------------------------------------------------------
Goal:
Instead of changing the read property directly every time,
create a prototype method.

Example idea:

Book.prototype.toggleReadStatus()

Its job:

Current status?
        ↓
Flip true ↔ false
        ↓
Return to click handler
        ↓
displayBooks()

Keeping behavior inside the Book object makes your code cleaner and easier to
maintain.

Review:
Objects & Object Constructors
https://www.theodinproject.com/lessons/node-path-javascript-objects-and-object-constructors

MDN - Object Prototypes
https://developer.mozilla.org/en-US/docs/Learn_web_development/Extensions/Advanced_JavaScript_objects/Object_prototypes


------------------------------------------------------------------------------
THE BIG IDEA
------------------------------------------------------------------------------

Never think:

Button clicked
    ↓
Edit the HTML

Instead think:

Button clicked
    ↓
Update myLibrary
    ↓
displayBooks()
    ↓
HTML is rebuilt from the array

The DOM is only a visual representation.

myLibrary is the real application state.

If your array is correct, displayBooks() can always recreate the entire UI.

This "data → render UI" pattern is the foundation of React, Vue, Angular, and
most modern frontend frameworks.
